    HTML/CSS classes

    Classes can be defined to demarcate almost any HTML element, like <p>, <button> or even an anchor tag, <a>.

    All classes in the <style> part or the CSS document must start with a dot: 

                
  .youtube {
    background-color: red;
    border: none;
  }

    Hover effects and transitions

    To create a hover effect for, say, a button, whose class name is YouTube, create a separate CSS entry with the syntax:

              
  .youtube:hover{
    background-color: blue;
  }

    This set of CSS properties activates only when being hovered.


    Transitions can be used to smoothen out hover effects and other purposes. To create a transition effect for the same button, set transition properties using the syntax:

              
  .youtube{
    transition: opacity 0.15s, background-color blue;
  }

    transition takes in pairs of parameters, the first one being the attribute to be changed followed by the value. To alter multiple attributes, separate the argument pairs with commas.


    NOTE: Transitions should always be placed under the main element CSS class and not the hover part. While counter-intuitive, this is required for smooth out-transitions after a cursor stops hovering.

    CSS box model

    Every HTML/CSS element has a box model, comprising of three separate properties - padding, margin and border.

    Padding refers to the spacing within the element, for example, the space between a button's text and its border.
    Margin refers to the spacing outside the element, and spaces elements from other elements.
    Border is the middle layer to padding and margin and it often the visible part.


    A box model for any element can be viewed by using developer tools on any browser.


    Text styles

    Understand basic text manipulation attributes like font-family, font-size, font-weight, font-style and text-align, most of them are intuitive.
    line-height is used to set spacing between lines.

    HTML entities

    Reserved characters in HTML must be replaced with character entities. HTML on its own does not support special characters so each of them has a numerical value preceded by &# and terminated with ; OR actual text like & gt; .

    A full list of HTML entities is available here.



    <p> or paragraph tags come with a top and bottom margin to separate previous/next paragraphs. <margin-top> and <margin-bottom> can be set to 0 to counter that.



    In case of multiple selectors (classes, divs, etc.) manipulating the same attribute, the selector with higher specificity takes priority. For example, 

            
  p {
    margin-bottom: 0px;
  }

  .stats{ /* an instance of <p> */
     margin-bottom: 20px;
  }

    The bottom margin gets set to 20px because .stats has higher specificity, i.e., class name selector > element selector.

    The id attribute has the highest specificity, meaning it overrides both inheritance and class names. Inline styles dominate over all CSS elements.


    To style in-line text, <span> tags come in handy. They can also be named by classes and styled.

    Working with hyperlinks

    <a> is an anchor tag used to specify links within texts or images.

            
  <a href="https://youtube.com/" target="_blank"> YouTube link </a>
    Pseudoclasses

    Pseudoclasses are used to target an element when it is in a specific state, for example, a link can be active, visited, or neither.

    a:link selector can be used to style idle links, a:visited for visited links and so on. The text-decoration property can be set to none to avoid ugly underlining of links.


    target is also an attribute of anchor tags than specified where the new link should open. Setting it to _blank opens the link in a new page.

    HTML structure 

              <!DOCTYPE html>

  <html>
  <head> </head>
  <body> </body>
  </html>

    is the barebones structure of a HTML document. <style> elements should be placed within <head>.

    Google Fonts provides HTML/CSS code to include texts not already available locally.

    Images and text boxes

    Images can be included with the following syntax:

    <img src="path/to/image">

    Images can also be a class for ease of styling.

    object-fit attribute provides sizing without having to change the dimensions of the image.


    It can take values like contain or cover. object-position is used to set which part of the image should be visible.



    Textboxes can be created using the input tag. 

          
  <input type="text"> <!--creates a textbox-->
  <input type="checkbox"> <!--creates a checkbox-->

    The placeholder attribute is used to show text in a textbox before it is interacted with.

    CSS Structure

    Types of elements in HTML

    1. element (eg. <p>) - they take up the entire line

    2. block element (eg. <img>, <input>) - takes only as much space as needed

    3. element - appears within a line of text


    Any element can be interconverted between the other three. A <p> class can be converted an inline block to increase space efficiency.

    The <div> element

    Probably the most important HTML element. Stands for division. Basically, it's just a box.

    <div> is a block element by default but can be changed using the display attribute.

    <div> serves as a container for all kinds of elements and hence is versatile.

    All block elements are confined to the <div> container when placed inside.

    Nested layouts

    Multiple <div> tags can be used to create concise vertical or horizontal layouts within a <div> itself. Since <div> is a block element, it can be left default for it to occupy the entire line, thereby creating a vertical layout.

    CSS grids

    Grids can be made by setting the display attribute to grid and using the grid-template-columns attribute to set column width for n number of grids.

    The same applies for the grid-template-rows attribute as well.

    Grids maintain alignment much better than inline blocks.

    Creating a higher number of rows than specified makes the new <div> element wrap around. 

              <div style="
display: grid;
grid-template-columns: 100px 100px 100px;
">
<div style="background-color: lightblue ;"> Div 1 </div>
<div style="background-color: lightpink;"> Div 2 </div>
<div style="background-color: lightblue;"> Div 3 </div>
<div style="background-color: lightpink;"> Div 4 </div>
</div>

    In the above example, we've created only 3 columns but creating a 4th one causes it to wrap around.




    Setting the column width to fr makes it occupy the entire line. fr = free space. Specifying a number before fr, like 1fr specifies the "ratio" in which they should occupy the space.


    column-gap attribute can be used to set the gap between columns of a grid (which is 0 by default). row-gap is intuitively similar.

    Flexbox

    Flexboxes behave like CSS grids but are more "flex"-ible. Flexboxes adjust their size as required. 

    <div style="
      display: flex;
      flex-direction: row;
      ">
     <div style="background-color: lightblue;"> div 1 </div>
     <div style="background-color: lightpink;"> div 2</div>
 </div>

    The flex attribute is the flexbox equivalent of fr in grids.

    Flexbox elements take their inherent width along with them, which provides for a much more flexible layout.


    justify-content is a POWERFUL attribute you can use to specify where the containers inside a flexbox go. This can be set to special values like space-between such that all the elements have equal space between them.

     <div style="
    display: flex;
    flex-direction: row;
    margin-top: 30px;
    height: 70px;
    border: 1px solid grey;
    justify-content: space-between;
    ">
    <div style="
    background-color: lightblue;
    width: 100px;
    ">
    div 1</div>
    <div style="
    background-color: lightpink;
    width: 100px;
    "> </div>
    <div style="
      background-color: lightblue;
      width: 100px;
    "> 
    </div>
  </div>

    align-items attribute can be used to specify how elements are aligned within a flexbox.

    Flexboxes can also be nested to create layouts easily.

    CSS position

    CSS has the following position attributes:

    • static

    • fixed

    • absolute

    • static

    Fixed

    Setting the position attribute to fixed anchors the element to the browser window, causing it to stay put even when the page is scrolled or resized.

    Absolute

    The absolute value also anchors the element, but to the page itself. This means it will appear at a position but will move up/down depending upon the position of the page relative to the browser window.

    Both fixed and absolute positions can be positioned with the properties top, bottom, left and right attributes.

    One special property of absolute position within a fixed position element is that using the above positioning attributes to position the absolute positioned element positions it within the parent element instead of the webpage itself.

    The same applies to absolute positioning within relatively positioned elements.

    z-index

    The z-index attribute sets how elements appear when multiple elements fixed or positioned absolutely appear on top of each other. By default, CSS elements definitely lower appear on front. The z-index property can be used to rectify that.
